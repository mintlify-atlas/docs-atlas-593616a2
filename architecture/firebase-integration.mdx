## Overview

HuelvaPedia uses **Firebase Realtime Database** as its backend for storing and retrieving encyclopedia content. The integration is real-time, meaning the UI automatically updates when database values change.

## Firebase Setup

### Database Reference Initialization

Both `MainActivity` and `Secundaria` initialize Firebase database references in `onCreate()`:

<CodeGroup>


<Info>
**MainActivity** always queries the `"principal"` node, while **Secundaria** dynamically queries based on the normalized category name.
</Info>

## Real-Time Listeners

### ValueEventListener Pattern

Both Activities use `ValueEventListener` to observe database changes:

```java MainActivity.java:50
reference.addValueEventListener(new ValueEventListener()
{
    @Override
    public void onDataChange(@NonNull DataSnapshot snapshot)
    {
        listaElementos.clear(); // Evita duplicaciГіn de datos cuando Firebase notifica cambios
        for (DataSnapshot dataSnapshot : snapshot.getChildren())
        {
            Elementos elemento = dataSnapshot.getValue(Elementos.class);
            listaElementos.add(elemento);
        }
        adaptador = new Adaptador(MainActivity.this, listaElementos);
        recyclerView.setAdapter(adaptador);
    }

    @Override
    public void onCancelled(@NonNull DatabaseError error)
    {
        Toast.makeText(MainActivity.this, "Algo ha fallado", Toast.LENGTH_SHORT).show();
    }
});
```

### Key Implementation Details

<Tabs>
  <Tab title="Data Retrieval">
    **onDataChange()** fires when:
    - Initial data is loaded
    - Any child node is added, changed, or removed
    
    ```java
    for (DataSnapshot dataSnapshot : snapshot.getChildren())
    {
        Elementos elemento = dataSnapshot.getValue(Elementos.class);
        listaElementos.add(elemento);
    }
    ```
    
    <Note>
    Firebase automatically deserializes JSON to Elementos objects using `getValue(Elementos.class)`.
    </Note>
  </Tab>
  
  <Tab title="Duplicate Prevention">
    **Clear before populating**:
    
    ```java MainActivity.java:55
    listaElementos.clear(); // Evita duplicaciГіn de datos cuando Firebase notifica cambios
    ```
    
    This ensures that when Firebase notifies of changes, items aren't duplicated in the list.
  </Tab>
  
  <Tab title="Error Handling">
    **onCancelled()** handles failures:
    

    <Info>

  </Tab>

## Data Synchronization

### Automatic UI Updates

Firebase listeners automatically update the UI when data changes on the server:

<Steps>
  <Step title="Database Change">
    Admin modifies data in Firebase Console
  </Step>
  
  <Step title="Listener Triggered">
    `onDataChange()` is called automatically
  </Step>
  
  <Step title="List Updated">
    ```java
    listaElementos.clear();
    for (DataSnapshot dataSnapshot : snapshot.getChildren()) {
        Elementos elemento = dataSnapshot.getValue(Elementos.class);
        listaElementos.add(elemento);
    }
    ```
  </Step>
  
  <Step title="UI Refreshed">
    ```java
    adaptador = new Adaptador(MainActivity.this, listaElementos);
    recyclerView.setAdapter(adaptador);
    ```
  </Step>
</Steps>

<Warning>
The app creates a **new adapter instance** on every update instead of calling `notifyDataSetChanged()`. While functional, this is less efficient for large datasets.
</Warning>

## Internet Connectivity Check

The splash screen (`PantallaCarga`) ensures internet connectivity before accessing Firebase:

```java PantallaCarga.java:62
private void checkConnectivity()
{
    final ConnectivityManager connectivityManager = (ConnectivityManager) getSystemService(Context.CONNECTIVITY_SERVICE);
    final NetworkInfo activeNetworkInfo = connectivityManager.getActiveNetworkInfo();

    hayConexion = activeNetworkInfo != null && activeNetworkInfo.isConnectedOrConnecting();

    if (hayConexion)
    {
        handler.postDelayed(navegarAPrincipal, DURACION_CARGA);
    }
    else
    {
        Toast.makeText(PantallaCarga.this, "SE DEBE DISPONER DE CONEXIГ“N A INTERNET", Toast.LENGTH_LONG).show();
        connectivityManager.registerNetworkCallback(
                new NetworkRequest.Builder()
                        .addCapability(NetworkCapabilities.NET_CAPABILITY_INTERNET)
                        .build(), connectivityCallback);
        monitorizando = true;
    }
}
```

### Network Callback

If no internet is detected, the app monitors for connectivity:

```java PantallaCarga.java:45
private final ConnectivityManager.NetworkCallback connectivityCallback = new ConnectivityManager.NetworkCallback()
{
    @Override
    public void onAvailable(Network network)
    {
        hayConexion = true;
        handler.postDelayed(navegarAPrincipal, DURACION_CARGA);
    }

    @Override
    public void onLost(Network network)
    {
        hayConexion = false;
        Toast.makeText(PantallaCarga.this, "SE DEBE DISPONER DE CONEXIГ“N A INTERNET", Toast.LENGTH_LONG).show();
    }
};
```

<Info>
The app **requires internet** to function since all content comes from Firebase Realtime Database.
</Info>

## Data Serialization

### Automatic Conversion

Firebase automatically converts between Java objects and JSON:

<Tabs>
  <Tab title="Firebase в†’ Java">
    **Deserialization** (Firebase to Elementos):
    
    ```java
    Elementos elemento = dataSnapshot.getValue(Elementos.class);
    ```
    
    **Requirements**:
    - Empty public constructor
    - Public getters/setters or public fields
    - Field names match Firebase keys
  </Tab>
  
  <Tab title="Java в†’ Firebase">
    **Serialization** (Elementos to Firebase):
    
    ```java
    // Not used in this app (read-only)
    // But would work like:
    databaseReference.setValue(elemento);
    ```
    
    Firebase would convert the Elementos object to JSON automatically.
  </Tab>
</Tabs>

## Read-Only Architecture

<Note>
HuelvaPedia is a **read-only** app. It only performs:
- `addValueEventListener()` - Read operations
- No `setValue()`, `updateChildren()`, or `removeValue()` calls

Content management happens through Firebase Console or admin tools.
</Note>

## Error Scenarios

| Scenario | Handling |
|----------|----------|
| No internet connection | `PantallaCarga` blocks navigation, shows toast |
| Firebase rules deny access | `onCancelled()` shows "Algo ha fallado" toast |
| Malformed data | Firebase returns null, app may crash without null checks |
| Wrong node name | Firebase returns empty snapshot, shows empty list |

<Warning>
The current implementation doesn't handle all error cases robustly. For example, if Firebase returns null for a field, calling methods on it could cause `NullPointerException`.
</Warning>

## Performance Considerations

### Current Approach

```java MainActivity.java:61
adaptador = new Adaptador(MainActivity.this, listaElementos);
recyclerView.setAdapter(adaptador);
```

**Drawbacks**:
- Creates new adapter on every Firebase update
- RecyclerView loses scroll position
- Less efficient for frequent updates

### Recommended Improvement

```java
// Initialize once in onCreate()
adaptador = new Adaptador(MainActivity.this, listaElementos);
recyclerView.setAdapter(adaptador);

// In onDataChange():
listaElementos.clear();
for (DataSnapshot dataSnapshot : snapshot.getChildren()) {
    listaElementos.add(dataSnapshot.getValue(Elementos.class));
}
adaptador.notifyDataSetChanged(); // Notify existing adapter
```

## Lifecycle Management

<Warning>
**Missing cleanup**: Neither Activity removes the Firebase listener in `onPause()` or `onDestroy()`, which could cause memory leaks.
</Warning>

**Recommended**:

```java
@Override
protected void onDestroy() {
    if (reference != null && valueEventListener != null) {
        reference.removeEventListener(valueEventListener);
    }
    super.onDestroy();
}
```

## Security Rules

While not shown in the code, Firebase Realtime Database requires security rules. For a read-only public app:

```json
{
  "rules": {
    ".read": true,
    ".write": false
  }
}
```

<Info>
This allows anyone to read data but prevents unauthorized writes.
</Info>
