## Overview

The `MainActivity` class is the entry point of the HuelvaPedia app. It displays the main categories of the encyclopedia in a RecyclerView, loads data from Firebase Realtime Database, and provides access to the weather feature.

<Note>
This activity loads data from the "principal" node in Firebase and automatically updates when the database changes.
</Note>

## Class Declaration

```java
package com.example.huelvapedia;

public class MainActivity extends AppCompatActivity
```

**Extends:** `AppCompatActivity`

## Firebase Integration

The activity uses Firebase Realtime Database to load encyclopedia entries:

<CodeGroup>

```java Data Loading Logic
reference = FirebaseDatabase.getInstance().getReference().child("principal");

reference.addValueEventListener(new ValueEventListener() {
    @Override
    public void onDataChange(@NonNull DataSnapshot snapshot) {
        listaElementos.clear(); // Prevents data duplication
        for (DataSnapshot dataSnapshot : snapshot.getChildren()) {
            Elementos elemento = dataSnapshot.getValue(Elementos.class);
            listaElementos.add(elemento);
        }
        adaptador = new Adaptador(MainActivity.this, listaElementos);
        recyclerView.setAdapter(adaptador);
    }
    
    @Override
    public void onCancelled(@NonNull DatabaseError error) {
        Toast.makeText(MainActivity.this, "Algo ha fallado", Toast.LENGTH_SHORT).show();
    }
});
```

## Navigation

The activity provides navigation to two destinations:

<Tabs>
  <Tab title="Secundaria Activity">
    When a user taps on a non-final entry (`ultimo = false`), the adapter navigates to `Secundaria` activity:
    
    ```java
    Intent intent = new Intent(v.getContext(), Secundaria.class);
    intent.putExtra("Nombre", elemento.getNombre());
    v.getContext().startActivity(intent);
    ```
    
    The entry name is passed as an extra to load the corresponding sub-category.
  </Tab>
  
  <Tab title="Tiempo Activity">
    When the weather button is clicked, navigates to the weather screen:
    
    ```java
    botonTiempo.setOnClickListener(v -> {
        Intent intent = new Intent(v.getContext(), Tiempo.class);
        startActivity(intent);
    });
    ```
  </Tab>
</Tabs>

```java Testing Firebase connection

    if (task.isSuccessful()) {
        Log.d("MainActivity", "Firebase connected successfully");
        Log.d("MainActivity", "Entries count: " + task.getResult().getChildrenCount());
    } else {
        Log.e("MainActivity", "Error loading data", task.getException());
    }
});
```

## Error Handling

<Expandable title="Firebase Error Handling">
  When Firebase fails to load data (network issues, permission errors, etc.):
  
  ```java
  @Override
  public void onCancelled(@NonNull DatabaseError error) {
      Toast.makeText(MainActivity.this, "Algo ha fallado", Toast.LENGTH_SHORT).show();
  }
  ```
  
  **Common causes:**
  - No internet connection
  - Firebase rules denying read access
  - Invalid database URL
  - Authentication issues (if rules require auth)
</Expandable>

<Expandable title="Data Duplication Prevention">
  The activity clears the list before adding new data to prevent duplicates when Firebase sends updates:
  
  ```java
  listaElementos.clear(); // Evita duplicación de datos cuando Firebase notifica cambios
  ```
  
  This is important because `ValueEventListener` fires whenever data changes, not just on initial load.
</Expandable>

## Lifecycle Considerations

<Warning>
The `ValueEventListener` remains active even when the activity is in the background. Consider removing the listener in `onStop()` or `onDestroy()` if you want to save resources:

```java
@Override
protected void onDestroy() {
    super.onDestroy();
    if (reference != null && valueEventListener != null) {
        reference.removeEventListener(valueEventListener);
    }
}
```
</Warning>
