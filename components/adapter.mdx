The Adaptador class is a custom RecyclerView adapter that bridges the data model (Elementos) with the view layer, handling both navigation items and actionable content with buttons.

## Overview

Adaptador extends `RecyclerView.Adapter<Adaptador.MiViewHolder>` and implements `View.OnClickListener` to manage the display of encyclopedia topics. It intelligently handles two types of items:

1. **Navigation Items**: Topics that lead to subtopic screens (Secundaria activity)
2. **Action Items**: Final-level content with interactive buttons (maps, phone, web links)

Creates and initializes the ViewHolder for each item in the RecyclerView.

<CodeGroup>

@NonNull
@Override
public MiViewHolder onCreateViewHolder(@NonNull ViewGroup parent, int viewType)
{
   View view = LayoutInflater.from(context).inflate(R.layout.principal, parent, false);

   return new MiViewHolder(view);
}

**Key Points:**
- Inflates the `R.layout.principal` layout for each item
- Attaches the adapter's click listener to the root view

<CodeGroup>

```java MiViewHolder
static class MiViewHolder extends RecyclerView.ViewHolder
{
    TextView nombre, descripcion;
    ImageView imagen;
    Button botonUbicacion, botonLlamar, botonEnlace;

    public MiViewHolder(@NonNull View itemView)
    {
        super(itemView);

        nombre = itemView.findViewById(R.id.tenombre);
        descripcion = itemView.findViewById(R.id.tedescri);
        imagen = itemView.findViewById(R.id.imagen);
        botonUbicacion = itemView.findViewById(R.id.botonubi);
        botonLlamar = itemView.findViewById(R.id.botonllamar);
        botonEnlace = itemView.findViewById(R.id.botonwiki);
    }
}
```

</CodeGroup>

## Behavior Logic

The adapter implements smart behavior based on the `ultimo` field of each Elementos object:

<Note>
The `ultimo` (last) boolean field determines whether an item is a navigation item or a final action item:
- **`ultimo = false`**: Item navigates to Secundaria activity with subtopics
- **`ultimo = true`**: Item displays action buttons (map, phone, web)
</Note>

### Navigation Items (ultimo = false)

- All action buttons are hidden
- Tapping the item launches Secundaria activity
- Passes the element's name as an Intent extra

### Action Items (ultimo = true)

- Item tap is disabled
- Action buttons are conditionally shown based on available data:
  - **Map Button**: Opens Google Maps with encoded location
  - **Phone Button**: Launches dialer with the phone number
  - **Web Button**: Opens the URL in a browser

## Key Features

### Image Loading with Picasso

Uses the Picasso library for efficient image loading and caching:

```java
Picasso.get().load(elemento.getFoto()).into(holder.imagen);
```

### Intent Actions

The adapter leverages Android's intent system for seamless integration:

- **ACTION_VIEW**: Opens maps and web browsers
- **ACTION_DIAL**: Initiates phone calls
- **createChooser()**: Allows users to select their preferred maps app

### Dynamic UI

Views are shown or hidden based on data availability:

```java
boolean tieneUbicacion = elemento.getUbicacion1() != null && elemento.getUbicacion2() != null;
boolean tieneTelefono = elemento.getTelefono() != null;
boolean tieneEnlace = elemento.getEnlace() != null;
```

### URL Encoding

Location data is properly encoded before passing to the maps intent:

```java
intent.setData(Uri.parse(elemento.getUbicacion1() + Uri.encode(elemento.getUbicacion2())));
```

## Dependencies

The Adaptador class relies on:

- **Android RecyclerView**: For list rendering
- **Picasso**: For image loading (`com.squareup.picasso:picasso`)
- **Elementos**: The data model class
- **Secundaria**: The secondary activity for navigation

<Note>
The adapter is reused across both MainActivity and Secundaria, demonstrating good code reuse practices. The same adapter logic handles different data sources by simply passing different contexts and element lists.
</Note>
